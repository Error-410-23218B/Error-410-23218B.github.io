Raytracing Project

In this post, we'll be exploring a CUDA-accelerated ray tracing renderer written in C++. Ray tracing is a rendering technique used to generate photorealistic images by simulating the way light behaves in the real world. It's a computationally intensive task, making it an ideal candidate for acceleration using NVIDIA's CUDA platform. We'll break down the code, explaining each component and how it contributes to the overall rendering process.

Includes and Definitions: The code starts with a list of includes for various libraries, including CUDA, OpenMP, and custom headers for vector math and ray tracing data structures.Error Handling: A custom error handling function, checkCudaErrors, is defined to handle CUDA errors.Device Functions: Several device functions are defined, including color, rand_init, render_init, render, create_world, and free_world. These functions will be executed on the GPU.Main Function: The main function is responsible for initializing the renderer, allocating memory, and launching the rendering kernel.

The color function is responsible for calculating the color of a ray as it interacts with the scene. It takes a ray object, a hitable list, and a curandState object as input. The function recursively calls itself to simulate the way light bounces around the scene, eventually returning the final color of the ray.

The rand_init function initializes a curandState object, which is used to generate random numbers on the GPU.

The render function is the core of the renderer. It takes a vec3 array (the framebuffer), image dimensions, number of samples per pixel, camera, world, and random state as input. The function launches a kernel that iterates over each pixel in the image, generating a ray for each pixel and calling the color function to calculate its color.

The create_world function creates a scene by allocating memory for a list of hitable objects and initializing them with random materials and positions.

The free_world function frees the memory allocated for the scene.
