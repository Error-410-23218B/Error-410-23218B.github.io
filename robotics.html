PID

    P-Proportional

    • Proportional decelerates the drive as it approaches the target
    • By calculating the error(distance from the target), we will multiply this by a gain factor so the closer the robot gets to its target, the smaller error will be
    • We will use this to decelerate the robot in drive commands
    • Equation : Kp e(t)

    I- Integral

    • Integral accelerates the drive when proportional undershoots.
    • Multiplies a gain factor by the total accumulation of errors over time
    • We will use this to accelerate the robot as it approaches the target to insure the robot reaches its destination
    • Equation : Ki e(t)

    D- Derivative

    • Derivative restricts rapid changes in motion for both acceleration and deceleration
    • Multiplies a gain factor by the change in distance from the target over the change of time
    • We will use this in our autonomous programs to prevent the robot from making sloppy movements that cause dri~ from the intended route
    • Equation: Kd de(t)⁄dt

Custom Drivetrain

The custom drivetrain uses the PID Controller class to control the limits and control the velocity and distance moved and degrees turned for the robot. Using a custom drivetrain instead of using the inbuilt vex drivetrain eliminates the inaccuracies in the vex drivetrain, which makes the robot‚Äôs autonomous more accurate and work more efficiently and it also means the driver control for the drivetrain is smoother and more intuitive to use.

The code is structured in an object oriented model, where the custom drivetrain inherits from the PID controller class, which is used for motion calculations, where there are various ‚Äúsetter‚Äù and ‚Äúgetter‚Äù methods to abstract the underlying class logic and function for the drivetrain making it easier to use in the rest of the program. The slewer algorithm also uses the time taken for the algorithm to set the max increase for the slewer.

The custom drivetrain also implements a ‚Äúslewer‚Äù algorithm which allows the drivetrain motors to synchronise together and to spin up to speed by quadratically interpolating the speed(‚Äúhalf and half again increments‚Äù). This ensures motors which are not mechanically linked spin at the same speed which ensures accurate motion control.

The custom drivetrain also implements odometry as the main use of the algorithm as it uses the absolute position, and using the aforementioned algorithms to drive towards the position on the field, where the orientation is changed using the turning algorithm, it changes using the PID algorithms for control and it spins the motor based on the amount of rotations required to turn the robot a certain amount.

RAMSETE Controller

The RAMSETE controller unlike other controllers is designed to actively profile and control the 2D motion of the robot, with the RAMSETE controller building on top of odometry it allows for the robot to accurately follow a desired trajectory while it prevents the robot from overshooting or straying from a set trajectory which makes it an accurate controller, where it represents the trajectory of the robot as a series of waypoints on the field and plotted against time.

It takes into account the actuators on a robot and how that affects the trajectory of the robot, it also unlike other controllers, which only control the robot‚Äôs local frame (e.g change in velocity or change in distance).

The RAMSETE controller via odometry is able to account for the robot‚Äôs global position and heading, which makes it a more viable controller for an accurate autonomous.
